Hello World 第一次使用 hexo 搭建属于自己的blog。

2019-12-26

为什么会 接触到 Hexo ？

 这个是我在项目里面遇到 需要做的 SEO搜索引擎优化，一位大佬介绍给我看的一个东西，有时候，有些东西表面看起来与需求不相关，但其实也是学习成长的一部分嘛。

 其实我有个想法，如果可以把 hexo 做成一个简单的产品网页的介绍 or 一个小公司的介绍。但其实要做的话其实还有更多的选择，所以嘛 虽然没能解决现阶段问题，我们可以先hold 一下，学习一下，来增加知识贮备。反正学到的是自己啊哈。

Hexo 好处

  • 免费！不需要花钱，就可以拥有，而且也不需要服务器和后台
  • 博客内容可以轻松打包、转移、发布
  • 可以随意绑定自己的域名，默认域名也不会失效，会自动跳转到新的域名
  • 等等

Hexo 安装

 其实不怎么想说安装，因为其实文档写的一清二楚了，So 我们来过一下安装流程吧。

安装前提:

 1.Node.js (Node.js 版本需不低于 8.10，建议使用 Node.js 10.0 及以上版本)

 //node -v 查看版本号

 2.Git(这个不用说了吧,按照文档走就好了。)

安装 Hexo：

$ npm install hexo

 可以用 命令行 hexo-v 查看安装是否成功，版本号

$ hexo -v

搭建blog

Github

 1.新建一个github账号(有的话直接忽略)

 2.github上新建一个仓库，仓库名字:username.github.io。
  例如我的账号名是 abson,那么仓库名就是 abson.github.io
  因为以后你可以直接访问 xxx.github.io 就可以看到你的blog 了

 3.生成SSH key
  3.1 回到您的git bash 上

$ git config --global user.name "yourname"
$ git config --global user.email "youremail"

  输入 用户名 和 邮箱
  可以由下面的两个命令查看你是否输入错误

$ git config user.name
$ git config user.email

  然后 创建 ssh key 

$ ssh-keygen -t rsa -C "youremail"

  就会生成SSH key ，你需要找到 .ssh 文件夹，找到id_rsa.pub文件，把里面的内容复制，然后到github 设置 添加 sshkey 密钥，title 随便写就好了

  如果你实在不知道这个文件在哪，你可以 git bash 输入命令,查看到大概位置

$   ~/. ssh

将hexo 部署到 github

  打开文件 配置文件 _config.yml
修改配置

 2.安装deploy-git，这样才能部署到github上

 然后【清理】-【生成】-【部署】

注意了 执行到最后部署可能需要让你输入账号密码

设置个人域名 （等待我的域名好了再上传）

基本上 搭建一个 自己blog 就已经完成，如果完成了以上步骤，那么就去浏览下 你的blog吧 网址是:xxx.github.io
